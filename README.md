# AIIS_final
人工智慧與資訊服務期末作品 - 自動化記帳及資料視覺化系統

## 專案說明
此專案為自動記帳及資料視覺化的專案

核心痛點：

> 記帳時需要使用額外的軟體
> 
> 視覺化時需要自行整理或輸出
>
> 報表無法準確呈現花費趨勢

為了解決上述的痛點本專案共分4個部分：

> 由捷徑輸入
> 
> 輸入至google試算表中
> 
> 使用html輸出現有的資料
> 
> 利用gemini API進行建議

此專案共有四個流程

>日常自動記帳
>
>資料視覺化
>
>預算調整
>
>AI檢查與提醒



## 記帳系統安裝說明如下：

> 本次專案採用google試算表及App Script作為主要運行環境

第一階段：準備 Google 試算表
開啟 Google 雲端硬碟，建立一個全新的 Google 試算表（或將既有的記帳範例存至你的雲端硬碟）。

將工作表（分頁）名稱命名為：記帳明細。

複製這家試算表的 ID：

    試算表 ID 位於瀏覽器網址列中，/d/ 之後到下一個斜槓 / 以前的一長串英數隨機字串（例如：1BJt6...Pk）。

-----------------------------------------------------------------------------------------------------------------------------

第二階段：設定 Google Apps Script (GAS)

在試算表上方工具列點選 「擴充功能」 > 「Apps Script」。

打開files中的app資料夾，分別將js html檔案等下載或複製

填入後端邏輯：將預設的 Code.gs 內容全部清空，並貼上你的 Code.gs.js 程式碼。

更換試算表 ID：在 Code.gs 內找到 SpreadsheetApp.openById('...')，將括號內貼上你剛剛複製的個人試算表 ID。

建立前端網頁：在編輯器左側點擊 「+」（新增檔案）> 選擇 「HTML」，並命名為 Chart（系統會自動產生 Chart.html）。

設計前端網頁：在編輯器左側點擊 「+」（新增檔案）> 選擇 「HTML」，並命名為 CSS。

讓前端網頁可以運作：在編輯器左側點擊 「+」（新增檔案）> 選擇 「HTML」，並命名為JS。

為使Appscript可以調用外部API，點擊左側的「專案設計」，勾選『在編輯器中顯示「appscript.json」資訊清單檔案』，貼上App中的json檔（appscript.jscon）。

點擊左側的「觸發條件」，選擇你要執行的項目「InstallableOnOpen」，選取活動來源「試算表」

點擊編輯器上方的 「儲存專案」 按鈕。

--------------------------------------------------------------------------------------------------------------------------------

第三階段：部署網頁應用程式 
點擊右上方藍色按鈕 「部署」 > 「新增部署」。

點選左側齒輪圖示，部署類型選取 「網頁應用程式」。

設定部署參數：

說明：可輸入「記帳專案」或自訂名稱。

執行身分：選擇 「我」（你的 Google 帳號）。

誰可以存取：務必選擇 「所有人」（Anyone），以便手機捷徑及瀏覽器可以順利存取。

點擊 「部署」。

首次授權存取：視窗會跳出需要授予權限，點擊「授予存取權」> 選擇你的 Google 帳號 > 點擊「進階」> 點擊「前往『記帳專案』（不安全）」> 點擊「允許」。

部署成功後，你會獲得一串網頁應用程式 URL（網址結尾為 /exec），請將其複製起來。

待完成部署後請在AppScript頁面中選擇「InstalledOnOpen」並點選執行，初次點擊時會需要授權。

## 記帳系統執行

>>> 本系統主要有以下四個前後端互動流程

1. 【流程 A】日常自動記帳

> 利用手機捷徑功能獲取記帳資料，並匯入至試算表中。
>
> 資料流：前端寫入花費明細-> 後端輸出

2. 【流程 B】資料視覺化及圖表互動

> 將試算表中的資料轉化為陣列資料並視覺化輸出。
> 利用javascript，本專案在使用時可以點擊任意圖表以調閱自己所需的資料。
>
> 資料流：後端調用現有資料表-> 前端輸出

3. 【流程 C】預算配置調整

> 前端網站中有更新預算的功能，點擊該功能可以覆寫預設的預算資料
>
> 資料流：前端-> 後端覆寫預算

4. 【流程 D】AI 財政盲審警示

> 在後端程式設立一個函式：AskGemini()，並透過該函式呼叫api以進行現有預算的評論及建議
>
> 資料流：後端-> Gemini模型 ->前端輸出評論
